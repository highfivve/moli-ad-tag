Interface: IMediaTypeVideo

For AdUnits with MediaType: video

See​

https://docs.prebid.org/dev-docs/adunit-reference.html#adunitmediatypesvideo

Properties​

api​

readonly api: Api[]

API frameworks supported

Values: 1: VPAID 1.0 2: VPAID 2.0 3: MRAID-1 4: ORMMA 5: MRAID-2 6: MRAID-3

Recommended

Example​

`[1, 2]`

See​

Prebid Server

battr?​

readonly optional battr: CreativeAttributes[]

Blocked creative attributes,

See​

OpenRTB 2.5 specification, List 5.3 for values

Example​

`[3, 9]`

context​

readonly context: "outstream" | "instream" | "adpod"

The video context, either 'instream', 'outstream', or 'adpod' (for long-form videos).

Defaults to ‘instream’

h?​

readonly optional h: number

Height of the video player in device independent pixels (DIPS) - Recommended

See​

[OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf]

linearity?​

readonly optional linearity: Linearity

Indicates if the impression is linear or nonlinear Values: 1: Linear/In-Stream 2: Non-Linear/Overlay.

See​

OpenRTB spec

maxbitrate?​

readonly optional maxbitrate: number

Maximum bit rate in Kbps.

See​

[OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf]

maxduration​

readonly maxduration: number

Maximum ad duration in seconds

See​

[OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf]

mimes​

readonly mimes: MimeType[]

Content MIME types supported.

Require by OpenRTB when using Prebid Server.

See​

https://www.iana.org/assignments/media-types/media-types.xhtml#video

Example​

`['video/mp4','video/x-flv']`

minbitrate?​

readonly optional minbitrate: number

Minimum bit rate in Kbps.

See​

[OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf]

minduration​

readonly minduration: number

Minimum ad duration in seconds

See​

[OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf]

placement​

readonly placement: Placement

Video placement type.

1: In-Stream Played before, during or after the streaming video content that the consumer has requested (e.g., Pre-roll, Mid-roll, Post-roll). 2: In-Banner Exists within a web banner that leverages the banner space to deliver a video experience as opposed to another static or rich media format. The format relies on the existence of display ad inventory on the page for its deliver 3: In-Article Loads and plays dynamically between paragraphs of editorial content; existing as a standalone branded message 4: In-Feed Found in content, social, or product feeds 5: Interstitial/Slider/Floating

Highly recommended because some bidders require more than context=outstream.

See​

OpenRTB 2.5 specification, List 5.9 for Values

playbackmethod​

readonly playbackmethod: PlaybackMethod[]

Allowed playback methods.. Defines whether inventory is user-initiated or autoplay sound on/off If none specified, all are allowed. For list, see [OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf].

Required by OpenRTB when using Prebid Server.

Values: 1: Auto-play, sound on 2: Auto-play, sound off 3: Click-to-play 4: mouse-over 5: Initiates on Entering Viewport with Sound On 6: Initiates on Entering Viewport with Sound Off by Default

playerSize​

readonly playerSize: undefined | [number, number] | [number, number][]

Player size(s) that this ad unit can accept (width, height).

plcmt​

readonly plcmt: Plcmt

See​

OpenRTB 2.6 specification, Plcmt Subtypes - Video

protocols​

readonly protocols: Protocol[]

Array of supported video protocols. For list, see [OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf].

Required by OpenRTB when using Prebid Server.

Supported video bid response protocols Values 1: VAST 1.0 2: VAST 2.0 3: VAST 3.0 4: VAST 1.0 Wrapper 5: VAST 2.0 Wrapper 6: VAST 3.0 Wrapper 7: VAST 4.0 8: VAST 4.0 Wrapper 9: DAAST 1.0 10: DAAST 1.0 Wrapper

Example​

`[5, 6]`

renderer?​

readonly optional renderer: IRenderer

The renderer associated to the ad-unit. Only for mediaType = video.

skip​

readonly skip: Skip

Indicates if the player will allow the video to be skipped, where 0 = no, 1 = yes.

If a bidder sends markup/creative that is itself skippable, the Bid object should include the attr array with an element of 16 indicating skippable video

startdelay​

readonly startdelay: number

Indicates the start delay in seconds for pre-roll, mid-roll, or post-roll ad placements.

Possible values:

0 Mid-Roll (value indicates start delay in second) 0 Pre-Roll -1 Generic Mid-Roll -2, Generic Post-Roll

See​

[OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf]

w?​

readonly optional w: number

Width of the video player in device independent pixels (DIPS) - Recommended

See​

[OpenRTB spec|https://www.iab.com/wp-content/uploads/2016/03/OpenRTB-API-Specification-Version-2-5-FINAL.pdf]
