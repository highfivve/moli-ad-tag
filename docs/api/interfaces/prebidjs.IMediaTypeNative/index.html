Interface: IMediaTypeNative

prebidjs.IMediaTypeNative

See

  • http://prebid.org/dev-docs/show-native-ads.html#native-ad-keys
  • https://docs.prebid.org/dev-docs/adunit-reference.html#adunitmediatypesnative

Properties

adTemplate

Optional Readonly adTemplate: string

Used in the AdUnit-Defined Creative Scenario, this value controls the Native template right in the page.

See

https://docs.prebid.org/prebid/native-implementation.html#3-prebidjs-native-adunit-overview

address

Optional Readonly address: IMediaTypeNativeRequirement

Address of the Buyer/Store. e.g, 123 Main Street, Anywhere USA

ad server key-value: hb_native_address

body

Optional Readonly body: IMediaTypeNativeRequirementWithLength

Text of the ad copy.

ad server key-value: hb_native_body

body2

Optional Readonly body2: IMediaTypeNativeRequirement

Additional Text of the ad copy.

ad server key-value: hb_native_body2

clickUrl

Optional Readonly clickUrl: IMediaTypeNativeRequirement

Where the user will end up if they click the ad.

ad server key-value: hb_native_linkurl

cta

Optional Readonly cta: IMediaTypeNativeRequirement

Call to Action text, e.g., Click here for more information.

ad server key-value: hb_native_cta

displayUrl

Optional Readonly displayUrl: IMediaTypeNativeRequirement

Text that can be displayed instead of the raw click URL. e.g, Example.com/Specials

ad server key-value: hb_native_displayUrl

downloads

Optional Readonly downloads: IMediaTypeNativeRequirement

The total downloads of the advertised application/product

ad server key-value: hb_native_downloads

icon

Optional Readonly icon: IMediaTypeNativeRequirementImage

The brand icon that will appear with the ad.

ad server key-value: hb_native_icon

image

Optional Readonly image: IMediaTypeNativeRequirementImage

A picture that is associated with the brand, or grabs the user's attention.

ad server key-value: hb_native_image

likes

Optional Readonly likes: IMediaTypeNativeRequirement

The total number of individuals who like the advertised application/product

ad server key-value: hb_native_likes

phone

Optional Readonly phone: IMediaTypeNativeRequirement

Phone Number of the Buyer/Store. e.g, (123) 456-7890

ad server key-value: hb_native_phone

price

Optional Readonly price: IMediaTypeNativeRequirement

The non-sale price of the advertised application/product

ad server key-value: hb_native_likes

privacyIcon

Optional Readonly privacyIcon: IMediaTypeNativeRequirement

Icon to display for the privacy link, e.g. http://example.com/privacy_icon.png

ad server key-value: hb_native_privicon

privacyLink

Optional Readonly privacyLink: IMediaTypeNativeRequirement

Link to the Privacy Policy of the Buyer, e.g. http://example.com/privacy

ad server key-value: hb_native_privacy

rating

Optional Readonly rating: IMediaTypeNativeRequirement

Rating information, e.g., 4 out of 5.

ad server key-value: hb_native_rating

rendererUrl

Optional Readonly rendererUrl: string

Used in the Custom Renderer Scenario, this points to javascript code that will produce the Native template.

See

https://docs.prebid.org/prebid/native-implementation.html#3-prebidjs-native-adunit-overview

Example

https://host/path.js

salePrice

Optional Readonly salePrice: IMediaTypeNativeRequirement

The sale price of the advertised application/product

ad server key-value: hb_native_saleprice

sendTargetingKeys

Optional Readonly sendTargetingKeys: boolean

Defines whether or not to send the hb_native_ASSET targeting keys to the ad server. Defaults to true for now, though we recommend setting this to false and utilizing one of the ways to define a native template.

See

https://docs.prebid.org/prebid/native-implementation.html#3-prebidjs-native-adunit-overview

sponsoredBy

Optional Readonly sponsoredBy: IMediaTypeNativeRequirement

The name of the brand associated with the ad.

ad server key-value: hb_native_brand

title

Optional Readonly title: IMediaTypeNativeRequirementWithLength

The title of the ad, usually a call to action or a brand name.

ad server key-value: hb_native_title

type

Optional Readonly type: "image"

Prebid.js defines types of native ad for you as a convenience. This way you have less code to maintain, that is hopefully more descriptive of your intent.

For now there is only the image type, but more will be added.

The image native ad type implies the following required fields:

  • image
  • title
  • sponsoredBy
  • clickUrl

And the following optional fields:

  • body
  • icon
  • cta

See

http://prebid.org/dev-docs/show-native-ads.html#pre-defined-native-types
