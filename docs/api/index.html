Building a publisher ad tag

Add at least the ad-tag as a dependency.

$ yarn add @highfivve/ad-tag

moli release script#

Moli provides a release script that helps you build releases and a summary page. Add the binary with:

$ yarn add @highfivve/moli-release

Authentication#

See github docs on authentication

Add the .npmrc file to publisher project, so that we can install the package from the github npm registry. As long as we have a private github repository we need to add the authToken (currently personal token).

@highfivve:registry=https://npm.pkg.github.com//npm.pkg.github.com/:_authToken={TOKEN}always-auth=trueregistry=https://registry.npmjs.org

I recommend putting this under ~/.npmrc as described in the npmrc documentation

Developing

You require yarn for developing.

yarn setup

If you have already checkout this repository you may want to clean things before:

yarn clean && yarn setup

Examples#

Unless you haven't just executed yarn setup, you need to build the ad tag and all modules first in order to run the examples.

yarn workspaces run make:nodemodule

Testing with examples#

If you want to test code changes within the examples you have to start the module in watch mode.

# rebuild the module you change, e.g. ad-tagyarn workspace @highfivve/ad-tag make:nodemodule --watch

Note that the workspace name for an example can be found in the respective project folder within the package.json.

And then start one of the example projects

yarn workspace @highfivve/example-publisher-mode start

Release

Releases are automatically prepared and publish via github actions. To trigger a release run

yarn lerna version

There will always be a release draft on the github release page. Pick the version the release drafter suggests.

After the version is bumped, push everything to the github repo. Assuming your remote is called github

git push github master --tags
